模态框（Modal）
模态框可以根据url动态将页面加载进来，或者使用原生的bootstrap模态框声明方式创建模态框
根据url动态加载页面S.popupOpen(url, [args])
在项目中，我们经常需要弹出框形式打开一些页面，用于便捷性的操作，比如用户编辑、用户信息浏览、类目选择器、图片选择器等，使用 S.popupOpen(url, [args])可以动态将需要加载的页面通过模态框打开。

使用模态框打开当前的模块页面

参数名 类型 默认值 说明
url String 打开的模块页面的url
args object 传递给模块页面的参数
定义打开的页面的meta的width属性值，可以声明打开的页面的宽度，如：modal.html的meta声明为:<meta width="500px" />
原生模态框
也许是chrome的bug，如果一个position:fixed元素的所有父元素中有一个元素有动画（即声明了css的animation），那么该元素的定位将不是根据body来获取定位的， 而是根据animation的元素进行定位，所以在此如果要用到原生的模态框，请将当前页面meta属性的animation置为none，代码如： <meta animation='none' />
还有未知的原因导致fixed定位失败，请尽量少用原生模态框，而是用S.popupOpen替代。
